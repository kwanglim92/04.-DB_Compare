## 1. 프로그램 소개

### 1.1 개요
DB_Compare는 Park Systems XE Service의 DB(데이터베이스) 설정값을 자동으로 검사하여 품질 관리를 지원하는 도구입니다.

### 1.2 주요 용도
- ✅ 장비 설정값이 규격에 맞는지 자동 검사
- ✅ 여러 장비의 설정 비교 및 검증
- ✅ QC 검사 결과를 Excel 리포트로 자동 생성
- ✅ 장비별 검사 기준(프로필) 관리

## 2. 시작하기

### 2.1 프로그램 실행

#### 방법 1: EXE 실행 (배포본, 권장)
1. 사내 공유 위치 또는 설치 경로에서 `DB_Compare_QC_Tool.exe` 더블클릭
2. 자동으로 메인 창이 표시됩니다 (Python 설치 불필요)

#### 방법 2: 명령 프롬프트에서 실행 (개발자)
1. Windows 검색창에서 "cmd" 입력
2. 명령 프롬프트 실행
3. 다음 명령어 입력:
```bash
cd "C:\Users\Spare\Desktop\03. Program\04. DB_Compare"
python main.py
```

#### 방법 2: 바로가기로 실행 (추천)
1. `main.py` 파일 우클릭
2. "연결 프로그램" → "Python"
3. 바로가기를 바탕화면에 생성하여 편리하게 사용

### 2.2 첫 실행 시 확인사항
- ✅ Python 3.11 이상이 설치되어 있는지 확인
- ✅ customtkinter 패키지 설치 확인
- ✅ DB 경로가 올바른지 확인 (`config/settings.json`)

## 3. 기본 사용법

### 3.1 QC 검사 수행 (4단계)

#### **STEP 1: DB 열기** 🗂️

1. 프로그램 상단의 **"Open DB"** 버튼 클릭
2. 파일 탐색기가 열리면 DB 폴더 선택
   ```
   일반적인 경로: C:\Park Systems\XEService\DB
   ```
3. "폴더 선택" 버튼 클릭
4. DB 로딩 완료 대기 (수초 소요)
5. 왼쪽 트리 뷰에 DB 구조가 표시됩니다

**💡 팁**: 
- DB 로딩 중 하단 상태표시줄에서 진행 상황 확인 가능
- 로딩이 완료되면 "DB Loaded. Ready for QC." 메시지 표시

---

#### **STEP 2: 프로필 선택** 📋

1. 툴바 중앙의 **프로필 드롭다운** 클릭
2. 검사할 프로필 선택 (예: `Test_Profile`)
3. 선택 즉시 아래 "Profile Viewer"에 검사 항목 목록 표시

**프로필이란?**
- 특정 장비나 용도에 맞는 검사 기준 세트
- 예: `NX10_Production`, `Test_Profile`, `Calibration_Check` 등

---

#### **STEP 3: QC 실행** ▶️

1. 상단의 **"Run QC"** 버튼 클릭
   - DB가 로딩되기 전에는 비활성화 상태입니다
2. 자동으로 모든 항목 검사 시작
3. 검사 완료 후 결과 자동 표시:
   - **오른쪽 상단**: 전체 통계 및 Pass Rate
   - **왼쪽 트리**: 항목별 Pass/Fail 색상 표시
   - **하단 테이블**: 상세 결과 및 실제 값

**결과 색상 의미**:
- 🟢 **초록색** PASS: 검사 통과 (스펙 범위 내)
- 🔴 **빨간색** FAIL: 검사 실패 (스펙 벗어남 / DB에 항목 없음)
- 🔵 **파란색** CHECK: 기록 전용 (합격/불합격 없음)
- ⚪ **회색** NO_SPEC: 검사 규격 없음
- 🟠 **주황색** ERROR: 검사 중 오류 (DB에서 값 읽기 실패)

**검증 방식 3가지** (프로필에서 항목별로 설정):

| 검증 타입 | 판정 기준 | 예시 |
|---|---|---|
| **Range** | `min ≤ 실제값 ≤ max` | ServoCutoffFrequency: `[60, 100]` Hz |
| **Exact** | `실제값 == 기대값` | FirmwareVersion: `"3.2.1"` |
| **Check** | 값만 기록 (판정 없음) | SerialNumber, Operator 등 |

---

#### **STEP 4: 리포트 저장** 💾

1. **"Export"** 버튼 클릭
2. 저장 위치 선택 (기본값: 바탕화면)
3. 파일명 확인/수정
   ```
   예: QC_Report_Test_Profile_2025-12-26_16-39-48.xlsx
   ```
4. "저장" 버튼 클릭
5. "Export Success" 메시지 확인

**리포트 내용**:
- **Summary 시트**: 전체 통계 및 차트
- **All Items 시트**: 전체 검사 항목 상세
- **Failed Items 시트**: 실패 항목만 필터링

---

### 3.2 트리 뷰 탐색

#### 기본 조작
- **클릭**: 항목 선택
- **더블클릭**: 폴더 펼치기/접기
- **Expand All**: 모든 항목 한번에 펼치기
- **Collapse All**: 모든 항목 한번에 접기

#### 트리 구조
```
Instrument: nx                           (최상위)
└─ Dsp                                   (모듈)
   ├─ XScanner                           (부품 타입)
   │  └─ 100um                           (부품 이름)
   │     ├─ ServoCutoffFrequencyHz       (검사 항목)
   │     ├─ ScanningRangeDefaultLow      (검사 항목)
   │     └─ ...
   └─ Info
      └─ General
         └─ Simulation
```

## 4. 프로필 관리

> **v1.3.0 변경 안내**: 기존 별도의 Profile Manager 창은 v1.3.0부터 **Admin 창의 "Spec 관리" 탭**으로 통합되었습니다. 아래 절차는 통합된 Admin 모드 기준입니다.

### 4.1 Admin 창 열기 (Spec 관리 진입)

1. 메인 툴바 우측의 **🔒 Admin** 버튼 클릭
2. 비밀번호 입력창이 나타납니다
3. 비밀번호 입력: **`pqc123`** (Admin 통합 비밀번호, v1.3+)
4. 확인 → 통합 Admin 창 오픈
5. 상단 **"Spec 관리"** 탭으로 이동

> 참고: Common Base 직접 편집(★ 모드)은 별도 비밀번호 `pqclevi`를 사용합니다.

**⚠️ 주의**:
- Spec 편집은 승인된 관리자만 가능합니다
- 비밀번호는 보안 유지를 위해 공유하지 마세요
- **오프라인 모드**에서는 자동으로 읽기 전용으로 전환됩니다

---

### 4.2 새 프로필 만들기

#### 단계별 가이드

**1단계: Profile Manager 열기**
- 비밀번호 입력 후 관리자 창 열기

**2단계: 새 프로필 생성**
1. 왼쪽 하단의 **"New"** 버튼 클릭
2. Profile Editor 창이 열립니다

**3단계: DB 선택** (이미 로딩된 경우 자동)
- DB가 로딩되지 않았다면 "Select DB" 버튼 클릭

**4단계: 검사 항목 선택**
1. 왼쪽 트리에서 검사할 항목 찾기
2. 체크박스 선택 (여러 개 가능)
3. 예시:
   ```
   ☑ Dsp → XScanner → 100um → ServoCutoffFrequencyHz
   ☑ Dsp → XScanner → 100um → ScanningRangeDefaultLow
   ```

**5단계: 검사 규격 설정**
1. 선택한 항목 **더블클릭**
2. Spec Config Dialog가 열립니다
3. 검사 방식 선택:

   **옵션 A: Range (범위 검사)**
   - 라디오 버튼: "Range" 선택
   - Min Spec: 최소값 입력 (예: 70)
   - Max Spec: 최대값 입력 (예: 90)
   - Unit: 단위 입력 (예: Hz)
   - 의미: 70 ~ 90 Hz 사이면 통과

   **옵션 B: Exact (정확값 검사)**
   - 라디오 버튼: "Exact" 선택
   - Expected Value: 기대값 입력 (예: Auto)
   - Unit: 단위 입력 (선택사항)
   - 의미: 정확히 "Auto"일 때만 통과

4. "OK" 버튼 클릭

**6단계: 프로필 저장**
1. 모든 항목 설정 완료 후 **"Save Profile"** 버튼 클릭
2. 프로필 이름 입력창이 나타납니다
3. 의미있는 이름 입력 (예: `NX10_Production_v2`)
4. "확인" 클릭
5. 성공 메시지 확인

---

### 4.3 프로필 수정

1. Profile Manager에서 수정할 프로필 선택
2. **"Edit"** 버튼 클릭
3. Profile Editor에서 항목 추가/삭제 및 규격 수정
4. 기존 항목 수정: 더블클릭 → 값 변경 → OK
5. 새 항목 추가: 체크박스 선택 → 더블클릭 → 규격 설정
6. **"Save Profile"** 클릭 (동일한 이름으로 덮어쓰기)

---

### 4.4 프로필 삭제

1. Profile Manager에서 삭제할 프로필 선택
2. **"Delete"** 버튼 클릭
3. 확인 대화상자: **"이 프로필을 정말 삭제하시겠습니까?"**
4. "예" 버튼 클릭
5. 프로필 목록에서 제거 확인

**⚠️ 경고**: 삭제된 프로필은 복구할 수 없습니다!

---

### 4.5 프로필 상속 (Advanced)

프로필은 다른 프로필을 상속받아 재사용할 수 있습니다.

#### 예시: Test_Profile이 Common_Base 상속
```json
{
  "base_profiles": {
    "Common_Base": {
      "Dsp": {
        "XScanner": {
          "100um": [
            {
              "item_name": "ServoCutoffFrequencyHz",
              "validation_type": "range",
              "min_spec": 70,
              "max_spec": 90,
              "unit": "Hz"
            }
          ]
        }
      }
    }
  },
  "equipment_profiles": {
    "Test_Profile": {
      "inherits_from": "Common_Base",
      "additional_checks": {
        "Dsp": {
          "Info": {
            "General": [
              {
                "item_name": "Simulation",
                "validation_type": "exact",
                "expected_value": "0"
              }
            ]
          }
        }
      }
    }
  }
}
```

**장점**:
- 공통 검사 항목을 한 번만 정의
- 여러 프로필에서 재사용
- 유지보수 용이

## 5. 고급 기능

### 5.1 뷰 모드 전환 (Profile Manager)

프로필 관리자에는 두 가지 뷰 모드가 있습니다:

#### Table View (테이블 뷰)
- 모든 항목을 플랫한 테이블로 표시
- 빠른 스캔 및 비교에 유용
- 7개 컬럼: Module, Part Type, Part, Item, Type, Spec, Unit

#### Tree View (트리 뷰)
- 계층 구조로 표시
- 구조 이해에 유용
- 3개 컬럼: Item, Specification, Unit

**전환 방법**:
- Profile Manager 상단의 토글 스위치 클릭
- 실시간으로 뷰 전환

---

### 5.2 검색 기능

프로필 관리자에서 빠르게 원하는 항목을 찾을 수 있습니다.

**사용 방법**:
1. Profile Manager 상단의 검색창 클릭
2. 키워드 입력 (예: "Servo", "Frequency")
3. 자동으로 일치하는 항목만 표시
4. 일치 항목은 주황색으로 강조
5. 검색 결과 개수 표시 (예: "2 of 5 items")

**검색 범위**:
- Module 이름
- Part Type 이름
- Part 이름
- Item 이름

---

### 5.3 테마 전환

프로그램은 Dark/Light 테마를 지원합니다.

**전환 방법**:
1. 우측 상단의 **"Dark Mode"** 토글 스위치 클릭
2. 즉시 테마 적용

**테마별 특징**:
- **Dark**: 눈의 피로 감소, 장시간 사용에 적합
- **Light**: 밝은 환경에서 가독성 향상

---

### 5.4 설정 파일 직접 편집

고급 사용자는 설정 파일을 직접 편집할 수 있습니다.

#### settings.json 위치
```
DB_Compare/config/settings.json
```

#### 편집 가능한 항목
```json
{
  "db_root_path": "C:\\Park Systems\\XEService\\DB",
  "default_equipment_profile": "",
  "ui_theme": "dark",
  "auto_save_reports": true,
  "report_output_dir": "./reports",
  "window_geometry": "1600x800"
}
```

**주의**: JSON 형식이 잘못되면 프로그램이 실행되지 않을 수 있습니다!

## 6. 자주 묻는 질문

### Q1: 프로그램이 실행되지 않아요
**A**: 다음을 확인하세요:
1. Python 3.11 이상이 설치되어 있는지
2. customtkinter 패키지가 설치되어 있는지
   ```bash
   pip install customtkinter
   ```
3. 명령 프롬프트에서 실행 시 오류 메시지 확인

---

### Q2: DB 로딩이 너무 오래 걸려요
**A**: 정상적인 현상입니다. DB 크기에 따라 수초~수십초 소요됩니다.
- 2,000개 이상의 항목: 약 10~20초
- 하단 상태표시줄에서 진행 상황 확인 가능

---

### Q3: Export 버튼이 비활성화되어 있어요
**A**: QC 검사를 먼저 실행해야 합니다.
1. "Open DB" → DB 로딩
2. "Run QC" → 검사 수행
3. "Export" 버튼 활성화됨

---

### Q4: 프로필을 삭제했는데 복구할 수 있나요?
**A**: 아니요, 삭제된 프로필은 복구할 수 없습니다.
- 중요한 프로필은 백업 권장
- `config/qc_specs.json` 파일을 정기적으로 백업하세요

---

### Q5: 검사 결과에서 "NO_SPEC"이 많이 나와요
**A**: 해당 항목에 대한 검사 규격이 설정되지 않았습니다.
- Profile Manager에서 프로필 편집
- 해당 항목에 규격 추가

---

### Q6: 여러 DB를 비교하고 싶어요
**A**: 현재 버전에서는 한 번에 하나의 DB만 로딩 가능합니다.
- 각 DB를 개별적으로 검사
- Export한 Excel 파일들을 비교

---

### Q7: Excel 리포트를 열 수 없어요
**A**: 다음을 확인하세요:
1. Microsoft Excel이 설치되어 있는지
2. 파일이 다른 프로그램에서 열려있지 않은지
3. 파일 경로에 한글이나 특수문자가 없는지

---

## 7. v1.5.0 신규 기능

### 7.1 DB 검색 (메인 창)

DB STRUCTURE 헤더에 검색 입력창이 있습니다.

**사용법:**
1. 검색창에 키워드 입력 (모듈명/Part/Item 모두 매칭)
2. 250ms 후 자동 검색 → 매치 항목 자동 펼침 + 노란 하이라이트
3. 활성 매치는 주황색으로 강조
4. **▲ / ▼ 버튼**으로 매치 간 이동
5. 카운트 라벨 `"3 / 12"` — 현재 위치 / 총 매치 수

**키보드 단축키:**
| 키 | 동작 |
|----|------|
| `Enter` (검색창 포커스) | 다음 매치로 이동 |
| `Shift+Enter` | 이전 매치로 이동 |
| `F3` | 다음 매치 (전역) |
| `Shift+F3` | 이전 매치 (전역) |
| `ESC` | 검색 클리어 |

검색을 클리어하면 검색 시작 전의 트리 펼침 상태로 자동 복원됩니다.

### 7.2 Open DB 개선 — 사내 네트워크 드라이브 지원

**개선사항:**
- 다이얼로그가 **마지막 사용 경로**에서 열림
- 권한 없는 모듈은 자동 skip (앱 멈춤 방지)
- 잘못된 폴더 선택 시 **명확한 안내** ("기대 구조: `<폴더>/Module/<ModuleName>/...`")
- **자동 정정**: 한 단계 위 폴더 선택 시 자동 탐색 → 확인 다이얼로그

**지원 경로 형식:**
- 로컬 경로: `C:\Users\...\DB`
- 매핑 드라이브: `Z:\share\DB`
- UNC 경로: `\\fileserver\share\DB`
- 한글/유니코드 경로 OK

VPN 연결이나 네트워크 드라이브 매핑이 안 되어 있으면 안내 다이얼로그가 표시됩니다.

### 7.3 Profile Viewer 결과 필터

QC 실행 후 PROFILE VIEWER 헤더의 **Segmented Button** `[ All | Pass | Check | Fail ]`로 필터링.

- 카운트 라벨 동적 갱신 (`"(12 of 156 items, FAIL)"`)
- 프로필 변경 시 자동 All로 리셋
- QC 미실행 상태에서 Pass/Check/Fail 클릭 시 안내 메시지

### 7.4 자동 업데이트 체크 (온라인 모드)

앱 시작 2초 후 자동으로 사내 서버에서 새 버전 확인.

**다이얼로그 동작:**
- 새 버전 발견 시 알림 + Release Notes 표시
- `[다운로드 페이지 열기]` `[나중에]` `[이 버전 무시]` 선택
- "이 버전 무시"는 settings.json에 누적 저장 — 이후 더 새 버전이 등장하면 다시 알림
- **강제 업데이트** 모드(보안 패치 등): "나중에"/"무시" 비활성, 다운로드만 가능

**수동 체크:** Admin → 서버 설정 탭 하단 `[지금 확인]` 버튼

오프라인 모드에서는 자동 체크가 동작하지 않습니다.

### 7.6 사내 매핑 드라이브가 동작하지 않을 때 (NPFS DRM 환경)

사내 보안 정책에 따라 일부 매핑 드라이브(예: `Z:`)는 **NPFS 기반 DRM 가상 볼륨**으로 마운트되어 있습니다. 이 경우 메모장으로는 파일이 열리지만 **DB_Manager(python.exe)에서는 `Permission denied (Errno 13)`** 가 발생합니다.

**진단 (PowerShell)**:
```powershell
Get-CimInstance Win32_LogicalDisk -Filter "DeviceID='Z:'" | Select FileSystem, ProviderName
```
- `FileSystem=NPFS` → 사내 DRM 적용 → 아래 우회 필요
- `FileSystem=NTFS, ProviderName=\\server\share` → 일반 SMB → 정상 동작 기대

**우회 방법: DRM 미적용 별도 SMB 서버 사용**:
1. 관리자가 DRM 미적용 별도 서버(예: `\\10.60.20.11\c.i.m\...`)에 동일 데이터 미러링
2. DB_Manager의 "Open DB"에서 그 UNC 경로를 선택
3. 표준 동작 — 코드 수정 불필요

**권장 사항**:
- 운영 환경에 따라 DB 폴더를 처음부터 별도 서버에 두는 워크플로우 정착 권장
- IT 부서에 DRM 화이트리스트(`python.exe`/`DB_Manager.exe`) 등록 요청도 가능하나 정책상 거부될 가능성 높음

## 9. v1.6 Industrial Checklist Auto-Fill (진행 중)

> **상태**: Phase 1~6 코드 구현 완료, 빌드 검증 + 사용자 검증 단계.

QC 검사 결과를 고객 출고용 Industrial Checklist 엑셀의 **G열(Measurement)** 에 자동으로 기입하는 기능입니다. 411행 규모의 엑셀에서 사람이 손으로 채우면서 발생하던 누락·오기 병목을 제거합니다.

### 9.1 5단 매핑 캐스케이드

DB의 dot-key(`Module.PartType.PartName.ItemName`)와 엑셀 C열의 자연어 항목명을 5단계로 시도하여 매핑합니다:

| 단계 | 방식 | 신뢰도 |
|---|---|---|
| **A. Explicit** | 엑셀 M열에 DB_Key가 직접 적힌 경우 | 1.00 |
| **B. Learned** | 이전에 수동 매핑한 학습 사전 hit | 0.95 |
| **C. Exact** | 정규화된 항목명 정확 일치 | 0.85 |
| **D. Fuzzy** | rapidfuzz 유사도 (≥ 0.80) | 0.80~ |
| **E. Unit hint** | 단위(H열) + Module 일치 | 0.60 |
| **Unmapped** | 미매핑 → top-5 후보 추천 | 0.0 |

### 9.2 안전 가드

- **백업 자동 생성**: `<원본>.bak.{timestamp}.xlsx`
- **G열만 쓰기**: A/I/F/M열 등 절대 건드리지 않음
- **수식 보호**: G33 LINEST 등 수식 셀 skip
- **그룹 헤더 skip**: D/E/G에 `-` 표시된 행 skip
- **표지 시트 무시**: 절대 쓰지 않음
- **Dry-run 기본 ON**: 첫 실행은 항상 미리보기
- **셀 색상 구분**: 녹색(자동/학습/정확) / 노란색(퍼지) / 빨간색(충돌)
- **`_AutoFill_Log` 시트**: 행/항목/키/신뢰도/출처/값/타임스탬프 자동 기록

### 9.3 사용 단계

1. QC 검사 완료 후 **Auto-Fill Checklist** 버튼 클릭 (QC 미실행 시 비활성)
2. 출고용 엑셀 파일 선택
3. 다이얼로그 통계 바에서 자동/학습/퍼지/미매핑 카운트 확인
4. 미매핑 항목 수동 매핑 (top-5 후보 드롭다운에서 선택)
5. **Dry-run 미리보기** 클릭 → 결과 리포트 + `_AutoFill_Log` 시트 확인
6. 검수 완료 시 **확정 기입** 클릭 → 백업 생성 + G열 실제 쓰기

### 9.4 학습 사전 누적

수동 매핑은 사내 PostgreSQL `checklist_mappings` 테이블에 저장됩니다. 다음 출고부터는 같은 모델의 같은 항목이 **Stage B (Learned)** 로 자동 매핑되어, 검수자의 수동 작업이 점차 줄어듭니다.

## 1. 프로그램 소개

### 1.1 개요
DB_Compare는 Park Systems XE Service의 DB(데이터베이스) 설정값을 자동으로 검사하여 품질 관리를 지원하는 도구입니다.

### 1.2 주요 용도
- ✅ 장비 설정값이 규격에 맞는지 자동 검사
- ✅ 여러 장비의 설정 비교 및 검증
- ✅ QC 검사 결과를 Excel 리포트로 자동 생성
- ✅ 장비별 검사 기준(프로필) 관리

## 3. 기본 사용법

### 3.1 QC 검사 수행 (4단계)

#### **STEP 1: DB 열기** 🗂️

1. 프로그램 상단의 **"Open DB"** 버튼 클릭
2. 파일 탐색기가 열리면 DB 폴더 선택
   ```
   일반적인 경로: C:\Park Systems\XEService\DB
   ```
3. "폴더 선택" 버튼 클릭
4. DB 로딩 완료 대기 (수초 소요)
5. 왼쪽 트리 뷰에 DB 구조가 표시됩니다

**💡 팁**: 
- DB 로딩 중 하단 상태표시줄에서 진행 상황 확인 가능
- 로딩이 완료되면 "DB Loaded. Ready for QC." 메시지 표시

---

#### **STEP 2: 프로필 선택** 📋

1. 툴바 중앙의 **프로필 드롭다운** 클릭
2. 검사할 프로필 선택 (예: `Test_Profile`)
3. 선택 즉시 아래 "Profile Viewer"에 검사 항목 목록 표시

**프로필이란?**
- 특정 장비나 용도에 맞는 검사 기준 세트
- 예: `NX10_Production`, `Test_Profile`, `Calibration_Check` 등

---

#### **STEP 3: QC 실행** ▶️

1. 상단의 **"Run QC"** 버튼 클릭
   - DB가 로딩되기 전에는 비활성화 상태입니다
2. 자동으로 모든 항목 검사 시작
3. 검사 완료 후 결과 자동 표시:
   - **오른쪽 상단**: 전체 통계 및 Pass Rate
   - **왼쪽 트리**: 항목별 Pass/Fail 색상 표시
   - **하단 테이블**: 상세 결과 및 실제 값

**결과 색상 의미**:
- 🟢 **초록색**: 검사 통과 (PASS)
- 🔴 **빨간색**: 검사 실패 (FAIL)
- ⚪ **회색**: 검사 규격 없음 (NO_SPEC)
- 🟠 **주황색**: 검사 중 오류 (ERROR)

---

#### **STEP 4: 리포트 저장** 💾

1. **"Export"** 버튼 클릭
2. 저장 위치 선택 (기본값: 바탕화면)
3. 파일명 확인/수정
   ```
   예: QC_Report_Test_Profile_2025-12-26_16-39-48.xlsx
   ```
4. "저장" 버튼 클릭
5. "Export Success" 메시지 확인

**리포트 내용**:
- **Summary 시트**: 전체 통계 및 차트
- **All Items 시트**: 전체 검사 항목 상세
- **Failed Items 시트**: 실패 항목만 필터링

---

### 3.2 트리 뷰 탐색

#### 기본 조작
- **클릭**: 항목 선택
- **더블클릭**: 폴더 펼치기/접기
- **Expand All**: 모든 항목 한번에 펼치기
- **Collapse All**: 모든 항목 한번에 접기

#### 트리 구조
```
Instrument: nx                           (최상위)
└─ Dsp                                   (모듈)
   ├─ XScanner                           (부품 타입)
   │  └─ 100um                           (부품 이름)
   │     ├─ ServoCutoffFrequencyHz       (검사 항목)
   │     ├─ ScanningRangeDefaultLow      (검사 항목)
   │     └─ ...
   └─ Info
      └─ General
         └─ Simulation
```

## 4. 프로필 관리

### 4.1 프로필 관리자 열기

1. 툴바의 **"Profile Manager"** 버튼 클릭
2. 비밀번호 입력창이 나타납니다
3. 비밀번호 입력: **`pqclevi`** (소문자)
4. 확인 버튼 클릭

**⚠️ 주의**: 
- 프로필 관리는 승인된 사용자만 가능합니다
- 비밀번호는 보안 유지를 위해 공유하지 마세요

---

### 4.2 새 프로필 만들기

#### 단계별 가이드

**1단계: Profile Manager 열기**
- 비밀번호 입력 후 관리자 창 열기

**2단계: 새 프로필 생성**
1. 왼쪽 하단의 **"New"** 버튼 클릭
2. Profile Editor 창이 열립니다

**3단계: DB 선택** (이미 로딩된 경우 자동)
- DB가 로딩되지 않았다면 "Select DB" 버튼 클릭

**4단계: 검사 항목 선택**
1. 왼쪽 트리에서 검사할 항목 찾기
2. 체크박스 선택 (여러 개 가능)
3. 예시:
   ```
   ☑ Dsp → XScanner → 100um → ServoCutoffFrequencyHz
   ☑ Dsp → XScanner → 100um → ScanningRangeDefaultLow
   ```

**5단계: 검사 규격 설정**
1. 선택한 항목 **더블클릭**
2. Spec Config Dialog가 열립니다
3. 검사 방식 선택:

   **옵션 A: Range (범위 검사)**
   - 라디오 버튼: "Range" 선택
   - Min Spec: 최소값 입력 (예: 70)
   - Max Spec: 최대값 입력 (예: 90)
   - Unit: 단위 입력 (예: Hz)
   - 의미: 70 ~ 90 Hz 사이면 통과

   **옵션 B: Exact (정확값 검사)**
   - 라디오 버튼: "Exact" 선택
   - Expected Value: 기대값 입력 (예: Auto)
   - Unit: 단위 입력 (선택사항)
   - 의미: 정확히 "Auto"일 때만 통과

4. "OK" 버튼 클릭

**6단계: 프로필 저장**
1. 모든 항목 설정 완료 후 **"Save Profile"** 버튼 클릭
2. 프로필 이름 입력창이 나타납니다
3. 의미있는 이름 입력 (예: `NX10_Production_v2`)
4. "확인" 클릭
5. 성공 메시지 확인

---

### 4.3 프로필 수정

1. Profile Manager에서 수정할 프로필 선택
2. **"Edit"** 버튼 클릭
3. Profile Editor에서 항목 추가/삭제 및 규격 수정
4. 기존 항목 수정: 더블클릭 → 값 변경 → OK
5. 새 항목 추가: 체크박스 선택 → 더블클릭 → 규격 설정
6. **"Save Profile"** 클릭 (동일한 이름으로 덮어쓰기)

---

### 4.4 프로필 삭제

1. Profile Manager에서 삭제할 프로필 선택
2. **"Delete"** 버튼 클릭
3. 확인 대화상자: **"이 프로필을 정말 삭제하시겠습니까?"**
4. "예" 버튼 클릭
5. 프로필 목록에서 제거 확인

**⚠️ 경고**: 삭제된 프로필은 복구할 수 없습니다!

---

### 4.5 프로필 상속 (Advanced)

프로필은 다른 프로필을 상속받아 재사용할 수 있습니다.

#### 예시: Test_Profile이 Common_Base 상속
```json
{
  "base_profiles": {
    "Common_Base": {
      "Dsp": {
        "XScanner": {
          "100um": [
            {
              "item_name": "ServoCutoffFrequencyHz",
              "validation_type": "range",
              "min_spec": 70,
              "max_spec": 90,
              "unit": "Hz"
            }
          ]
        }
      }
    }
  },
  "equipment_profiles": {
    "Test_Profile": {
      "inherits_from": "Common_Base",
      "additional_checks": {
        "Dsp": {
          "Info": {
            "General": [
              {
                "item_name": "Simulation",
                "validation_type": "exact",
                "expected_value": "0"
              }
            ]
          }
        }
      }
    }
  }
}
```

**장점**:
- 공통 검사 항목을 한 번만 정의
- 여러 프로필에서 재사용
- 유지보수 용이

## 5. 고급 기능

### 5.1 뷰 모드 전환 (Profile Manager)

프로필 관리자에는 두 가지 뷰 모드가 있습니다:

#### Table View (테이블 뷰)
- 모든 항목을 플랫한 테이블로 표시
- 빠른 스캔 및 비교에 유용
- 7개 컬럼: Module, Part Type, Part, Item, Type, Spec, Unit

#### Tree View (트리 뷰)
- 계층 구조로 표시
- 구조 이해에 유용
- 3개 컬럼: Item, Specification, Unit

**전환 방법**:
- Profile Manager 상단의 토글 스위치 클릭
- 실시간으로 뷰 전환

---

### 5.2 검색 기능

프로필 관리자에서 빠르게 원하는 항목을 찾을 수 있습니다.

**사용 방법**:
1. Profile Manager 상단의 검색창 클릭
2. 키워드 입력 (예: "Servo", "Frequency")
3. 자동으로 일치하는 항목만 표시
4. 일치 항목은 주황색으로 강조
5. 검색 결과 개수 표시 (예: "2 of 5 items")

**검색 범위**:
- Module 이름
- Part Type 이름
- Part 이름
- Item 이름

---

### 5.3 테마 전환

프로그램은 Dark/Light 테마를 지원합니다.

**전환 방법**:
1. 우측 상단의 **"Dark Mode"** 토글 스위치 클릭
2. 즉시 테마 적용

**테마별 특징**:
- **Dark**: 눈의 피로 감소, 장시간 사용에 적합
- **Light**: 밝은 환경에서 가독성 향상

---

### 5.4 설정 파일 직접 편집

고급 사용자는 설정 파일을 직접 편집할 수 있습니다.

#### settings.json 위치
```
DB_Compare/config/settings.json
```

#### 편집 가능한 항목
```json
{
  "db_root_path": "C:\\Park Systems\\XEService\\DB",
  "default_equipment_profile": "",
  "ui_theme": "dark",
  "auto_save_reports": true,
  "report_output_dir": "./reports",
  "window_geometry": "1600x800"
}
```

**주의**: JSON 형식이 잘못되면 프로그램이 실행되지 않을 수 있습니다!

## 6. 자주 묻는 질문

### Q1: 프로그램이 실행되지 않아요
**A**: 다음을 확인하세요:
1. Python 3.11 이상이 설치되어 있는지
2. customtkinter 패키지가 설치되어 있는지
   ```bash
   pip install customtkinter
   ```
3. 명령 프롬프트에서 실행 시 오류 메시지 확인

---

### Q2: DB 로딩이 너무 오래 걸려요
**A**: 정상적인 현상입니다. DB 크기에 따라 수초~수십초 소요됩니다.
- 2,000개 이상의 항목: 약 10~20초
- 하단 상태표시줄에서 진행 상황 확인 가능

---

### Q3: Export 버튼이 비활성화되어 있어요
**A**: QC 검사를 먼저 실행해야 합니다.
1. "Open DB" → DB 로딩
2. "Run QC" → 검사 수행
3. "Export" 버튼 활성화됨

---

### Q4: 프로필을 삭제했는데 복구할 수 있나요?
**A**: 아니요, 삭제된 프로필은 복구할 수 없습니다.
- 중요한 프로필은 백업 권장
- `config/qc_specs.json` 파일을 정기적으로 백업하세요

---

### Q5: 검사 결과에서 "NO_SPEC"이 많이 나와요
**A**: 해당 항목에 대한 검사 규격이 설정되지 않았습니다.
- Profile Manager에서 프로필 편집
- 해당 항목에 규격 추가

---

### Q6: 여러 DB를 비교하고 싶어요
**A**: 현재 버전에서는 한 번에 하나의 DB만 로딩 가능합니다.
- 각 DB를 개별적으로 검사
- Export한 Excel 파일들을 비교

---

### Q7: Excel 리포트를 열 수 없어요
**A**: 다음을 확인하세요:
1. Microsoft Excel이 설치되어 있는지
2. 파일이 다른 프로그램에서 열려있지 않은지
3. 파일 경로에 한글이나 특수문자가 없는지

---

### Q9: 프로그램이 갑자기 종료되었어요
**A**: 오류 로그를 확인하세요:

2. 스크린샷과 함께 개발자에게 보고
3. DB 경로나 권한 문제일 가능성이 높습니다
